# Contributing

Thanks for taking the time to look at this project.

Sample Pack Assembler is a personal tool released publicly for other producers to use. Contributions are welcome in a limited scope given the restrictive license. Here is what you need to know before you open anything.

---

## What is welcome

**Bug reports** — if something crashes, misbehaves or produces unexpected output, open an issue with a clear description of what happened, your OS, your Python version, and the steps to reproduce it. The more precise you are, the faster it gets fixed.

**Feature requests** — if you have an idea that would make the tool genuinely better for producers, open an issue and describe the use case. Not every request will be implemented but all of them are read.

**Typos and documentation fixes** — small corrections to the README, CHANGELOG or inline comments are fine as a pull request without prior discussion.

---

## What is not welcome

Pull requests that add new dependencies beyond `pywebview` without prior discussion.

Pull requests that change the visual design of the themes without being invited to do so.

Anything that would conflict with the license terms, including attempts to make the application callable as a library or embeddable in other software.

---

## Pull requests

Open an issue first for anything beyond a typo fix. Once the direction is agreed on, fork, branch off `main`, make your changes, and open a PR with a clear description of what changed and why.

Keep commits clean. One commit per logical change. No "wip" commits in the final PR.

---

## Code style

The project has no linter configuration. Match the style of the file you are editing: 4-space indentation, descriptive variable names, comments only where the logic is not self-evident.

---

## Respect the license

Any contribution you submit is understood to be granted under the same license as the rest of the project. If you are not comfortable with that, do not contribute.
